## 1. 🧠 基本理念と概要

- すべての要素は「token（語）」であり、**名詞でも動詞でもある**
    
- 品詞の明示的分類はない（使用位置と関係で意味を解釈）
    
- 自然言語における SVO（主語・動詞・目的語）を基本としつつ、**自由な順序や修飾も許容**
    
- プログラマ的発想（関数、バグ、デバッグ、クラッシュなど）が中心語彙
	
- 世界中のプログラマが簡単にフィーリングで会話ができるのが目標

---

## 2. 🧱 文の構造

文は原則として「**tokenの連なり**」で意味を構成。  
文末には `;` をつける。

### ✅ 最小構成：

`[主語] [動作] [対象] ;`

### ✅ 例：

`mi knw fnk;`

→ 「私は関数を理解している」

## 3. 🔤 トークンの種類

### 3.1 主体トークン（代名詞・主語）

|トークン|意味|
|---|---|
|`mi`|私（1人称）|
|`ra`|あなた（2人称）|
|`shs`|彼/彼女（3人称単）|
|`thy`|彼ら（3人称複）|

---

### 3.2 動作・状態トークン（動詞的）

|トークン|意味|
|---|---|
|`knw`|理解する|
|`hlp`|助ける|
|`fid`|見つける|
|`wnt`|欲しい|
|`gipush`|努力する|
|`wrt`|書く|
|`lok`|見る|
|`dgb`|デバッグする|
|`clash`|壊れる / クラッシュする|
|`staka`|詰まる / スタックする|

### 3.3 対象・物体トークン（名詞的）

|トークン|意味|

|`fnk`|関数|

### 3.4 修飾・副詞トークン

| トークン   | 意味    |
| ------ | ----- |
| `bit`  | 少し    |
| `int`  | まあまあ  |
| `lng`  | 超〜    |
| `no`   | 否定    |
| `rua~` | 再び    |
| `~fy`  | 〜化する  |
| `by~`  | 〜によって |

---

### 3.5 機能・構文トークン

| トークン   | 意味               |
| ------ | ---------------- |
| `d`    | 過去形（時制）          |
| `ig`   | 進行形（〜している）       |
| `if~`  | もし〜なら            |
| `lse~` | そうでないなら          |
| `.`    | 間（ポーズ 0.25~1sec） |
| `;`    | 文末マーク            |
| `()`   | 入れ子（括弧構文）        |

---

### 3.6 メタ・言語トークン

| トークン  | 意味            |
| ----- | ------------- |
| `nha` | 〜についての（語順：前置） |
| `-:~` | ーで言う～（翻訳ラベル）  |
| `wt`  | what（何）       |
| `whr` | where（どこ）     |
| `wy`  | why（なぜ）       |
| `whn` | when（いつ）      |
| `hw`  | how（どうやって）    |
| `wtf` | 何だこれは!?（感嘆）   |

## 4. ⚙️ 構文規則

### 4.1 SVO構文（基本）

`mi knw fnk; → 私は関数を理解している。`

### 4.2 修飾構造

`ra ig gipush; → あなたはめっちゃ努力している。`

`mi no knw koda; → 私はコードを理解していない。`

---

### 4.3 否定・条件・再帰

`mi dbg if mi fid bg; lse... → もし私がバグを見つけたら、デバッグする。そうでなければ...`

`ra rua pblsh d koda; → あなたはコードを再公開した。`

---

### 4.4 入れ子構造（括弧）

`ig lok (fnk nha bg); → バグについての関数を見てる。`

`mi cn wrt (nm k fnk); → 私は関数の名前を書くことができる。`

---

### 4.5 言語指定・翻訳表現

`hlp i japanese:(助ける); → "hlp" は日本語で "助ける"`

`nm english:(debug) i dgb; → 英語で言う"debug" は "dbg"`

## 5. ✍️ 例文集（ナチュラル文⇄grk-lang）

| 日本語文          | grk-lang                |
| ------------- | ----------------------- |
| 私は関数を理解している   | `mi knw fnk;`           |
| あなたはバグをデバッグした | `ra dgb d bg;`          |
| 彼らは今スタックしている  | `thy ig staka;`         |
| もし私がバグを見つけたら… | `if mi fid bg...;`      |
| 「助ける」は日本語で何？  | `wt nm (japanese):hlp?` |

---

## 6. 🧠 設計指針と応用

- 言語は**人間の意思表現**＋**コード的操作性**の中間に位置
    
- 文は評価される命令であり、記述でもある
	
- 形式はすごく緩いので、母語によって、方言的なものもできる
